📋 The Brief

Take a set of complex 3D mechanical assemblies — shafts, helical gears, bearing housings, fasteners, and multi-part joints — and annotate every component with precision labels that an AI model could learn from. These weren't simple shapes; the assemblies included interlocking parts that looked similar from certain angles but were functionally completely different.

🔧 The Process

Step 01
Model Intake & Review
Each assembly opened in FreeCAD, reviewed in full — rotating the model, inspecting the part tree, understanding component relationships before labelling.
Step 02
Label Schema Design
Established consistent label taxonomy: SHAFT_xx for rotational elements, GEAR_xx for transmission parts, JOINT_xx for connection points, ASSEMBLY_vx for version tracking.
Step 03
Component Annotation
Each part selected within FreeCAD's model tree and assigned its label. Material properties, dimensional constraints, and spatial relationships recorded alongside each tag.
Step 04
Edge Case Handling
Overlapping components required sub-labels to distinguish parent/child relationships without breaking the annotation schema.
Step 05
Quality Review
Every annotated assembly reviewed against original specification. Mislabelled parts, missing tags, and inconsistent naming corrected before export.
Step 06
Export & Handoff
Final models exported in AI-compatible formats with label manifests for each assembly version.

📋 The Brief

Take raw structural BIM models representing multi-storey structural frames and apply semantic labels to every element so AI systems could learn to identify structural components from 3D building data. Each element needed the correct IFC class, structural role documentation, and property set data.

🔧 The Process

Step 01
IFC Model Import
Models imported into BlenderBIM/Bonsai from IFC format. Element tree reviewed to identify unlabelled or mislabelled components.
Step 02
Element Classification
Every structural element assigned its correct IFC class — IfcBeam, IfcColumn, IfcSlab, IfcWall — based on structural function and geometry.
Step 03
Property Set Assignment
Structural properties assigned to each element: material, load-bearing status, fire rating, and dimensional data embedded as IFC property sets.
Step 04
Spatial Hierarchy
Every element verified to sit in the correct IFC spatial container: Site → Building → Floor → Element.
Step 05
Clash Detection
Elements checked for spatial clashes — unexpected column/slab intersections, misaligned beams — and corrected before export.
Step 06
IFC Validation
Final IFC files validated against IFC schema to confirm all labels, properties, and hierarchies were correctly formed.

📋 The Brief

Object detection is one of the foundational tasks in computer vision. Before any model can identify what's in an image, a human has to teach it. The task involved preparing large-scale bounding box annotation datasets for object detection models — images containing industrial components and assembly elements that the model needed to learn to detect and classify precisely.

🔧 The Process

Step 01
Batch Review
Each image batch reviewed before annotation — checking quality, lighting consistency, and object visibility. Blurred images flagged and removed.
Step 02
Class Schema Setup
Class taxonomy defined per project — object types to detect, handling overlapping objects, and minimum confidence thresholds for partial visibility.
Step 03
Bounding Box Drawing
Tight bounding box drawn for each object — as close to the boundary as possible without clipping. Coordinates recorded in [x, y, width, height] format.
Step 04
Class Label Assignment
Each box assigned its class label. Ambiguous objects flagged for review rather than guessed — accuracy over speed.
Step 05
Confidence Scoring
Confidence score assigned to each annotation — reflecting how clearly visible the object was. Low-confidence annotations reviewed by second pass.
Step 06
QA & Export
Complete dataset reviewed for missed objects, incorrect boxes, and class inconsistencies before final export for model training.

📊 Annotation Format

{ "image_id": "frame_0042", "annotations": [ { "label": "PART_A", "confidence": 0.94, "bbox": [40, 38, 70, 55], "format": "[x, y, width, height]", "occluded": false }, { "label": "PART_B", "confidence": 0.87, "bbox": [140, 50, 80, 65], "occluded": true, "occlusion_pct": 15 } ] }

📋 The Brief

Point cloud annotation sits at the frontier of AI training data work. Unlike images, point clouds are raw 3D spatial data — millions of individual points captured by LiDAR or structured light sensors, each with an x, y, z coordinate. The task was to identify meaningful object clusters, assign class labels to each, and document spatial boundaries precisely enough for autonomous perception models to learn from.

🔧 The Process

Step 01
Scan File Loading
Raw .pcd and .las scan files loaded and visualised in 3D. Initial inspection assessed point density, scan coverage, and presence of noise or artefacts.
Step 02
Noise Filtering
Statistical outlier removal applied to clean the point cloud — eliminating stray points that would contaminate the annotation.
Step 03
Cluster Segmentation
Point clusters segmented — isolating spatially distinct groups corresponding to individual objects. Density-based segmentation as primary method.
Step 04
Class Label Assignment
Each cluster assigned its class label based on geometric profile and spatial context — no improvised classes, no guesses.
Step 05
Boundary Documentation
3D bounding volumes fitted around each labelled cluster. Boundary coordinates recorded per cluster.
Step 06
QA Review
Annotated scans reviewed from multiple viewpoints — checking for missed clusters, label errors, and boundary fit quality.
